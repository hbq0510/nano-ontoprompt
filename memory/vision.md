## 终极场景

构建一个**基于Flow（数据流）的防空反导智能推演平台**，能够处理饱和攻击下的多层拦截调度。

核心参考文档：`桌面/简化例子_修改.md` 和 `桌面/反导拦截调度本体智能场景2026_07_05_v1.1.docx`

## 核心概念

1. **Flow 模型**：不仅仅是实体（雷达、拦截弹、威胁），更重要的是它们之间的**关系流**——探测流、火力通道流、威胁流。每条流有自己的属性（信噪比、拦截时间窗口、累积杀伤概率等），推演本质是**对这些流进行动态编织和调度**。

2. **时钟模型**：当前采用离散 tick（1 tick ≈ 10秒），后续可能需要变速时钟（普通阶段10s、威胁阶段1s、拦截阶段0.1s）。

3. **情报插入循环**：推演中途可以插入自由文本情报 → LLM 解析成结构化操作（修改 Link 属性、创建新 Link、重分配资源）→ 推演循环继续。这是区别于传统确定性仿真的核心价值。

4. **多层拦截概率聚合**：系统需要沿 Threat→FireChannel→Threat→FireChannel→HVA 路径进行 `Agg.Product()` 等聚合计算，得出最终的突防概率。

5. **LLM 的两层作用**：(1) 解析情报文本为结构化操作；(2) 智能生成拦截方案。本体约束防止幻觉。
